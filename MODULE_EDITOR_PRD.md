# FORMA Live 模块编辑页设计模式 (PRD)

## 1. 概述 (Overview)
本文档旨在梳理并统一 FORMA Live 中所有**核心产物模块**（如：讲解视频、智能问卷、研究报告等）的编辑详情页 UI/UX 设计模式。
基于“讲解视频”编辑页的现有优秀实践，通过抽象其核心结构，为其他模块提供标准的搭建规范（Pattern），确保整个产品在多模块扩展时具备高度一致的用户体验。

---

## 2. 核心设计理念 (Core Philosophy)
本设计模式的核心在于**“渐进式展开（Progressive Disclosure）”**与**“AI 伴随式创造（Copilot-driven Creation）”**：
1. **产物与素材分离**：将“最终交付物”与“构建该交付物的底层碎片素材”在顶层路由上进行物理隔离，降低信息噪音。
2. **全局一致的 AI 副驾**：无论用户在查看最终视频，还是在逐字修改大纲，AI 助手始终在右侧固定位置提供上下文感知的操作支持。
3. **同构的编辑空间**：针对不同类型的数据（文本、结构化列表、多媒体），提供标准化的画布切换逻辑，不改变页面的整体骨架。

---

## 3. 标准页面布局框架 (Standard Layout Dashboard)

所有模块的编辑详情页必须遵循以下基础分层结构：

### 3.1 顶层全局导航栏 (Global Header)
页面最上方固定栏，提供全局元信息与核心业务流转操作：
- **左侧**：`[返回图标] [模块/项目名称 (支持点击重命名编辑 ✏️)]`
- **右侧**：`[语言切换(中/文)] [全屏预览按钮] [下载产物按钮] [🚀 发布按钮 (黑色主视点)]`

### 3.2 顶层状态与视图切换栏 (View & Status Bar)
导航栏下方，用于控制主控台的三大核心视图（Views），替代了原本在画布内部切换 Tab 的复杂逻辑。
- **左侧视图切换器 (Views)**：
  - **Tab 1: 最终产物态 (例如：智能课程 / 智能问卷 / 智能报告)** -> 即“终态”预览。
  - **Tab 2: 组成素材态 (例如：课程素材 / 问卷素材 / 文档素材)** -> 即“过程与物料”管理。
  - **Tab 3: 制作过程与日志 (例如：制作过程)** -> 提供操作历史或系统生成日志。
- **右侧状态提示 (Status Banner)**：
  - 显示当前模块的整体完毕状态，例如：`🪄 已经制作了智能课程，请您查看 🎉`。

### 3.3 主控工作台 (Workspace Canvas)
根据上方 Tab 的切换，主控工作台会呈现 2 种标准响应式布局（右侧的 Copilot 永远固定存在）。

#### 模式 A：最终产物态 (Tab 1 选中时)
- **布局特点**：`双栏布局 (2-Column)`。隐藏左侧素材目录树，最大化展示最终成果。
- **主画布 (Main Area)**：
  - 顶部显示标签（如浅绿色底色的“讲解视频”）。
  - 下方为主播放器、幻灯片最终预览、或问卷的沉浸式预览。
  - 包含该产物特有的悬浮控件（如“目录”按钮）。
- **右侧面板**：`Copilot 对话窗口`。

#### 模式 B：素材态 (Tab 2 选中时)
- **布局特点**：`三栏布局 (3-Column)`。展开素材维度的导航树。
- **左栏 (Menu/Asset Tree)**：
  - 该模块的所有原材料与中间过程产物的列表（树状或平铺）。
  - **分组建议**：按生成链路排序，如：`输入源` (原文档/解析内容) -> `中间层` (大纲/脚本/逻辑树) -> `媒体片段` (幻灯片/语音/封面图)。
- **中栏 (Detail Canvas)**：
  - 具体素材的专属阅读/编辑区。**此区域是多态的**，必须根据左侧选中的类型加载对应的 Editor。
  - *如果是文本/列表型（如大纲、脚本）*：显示完整的结构化富文本内容。
  - *如果是视觉片段型（如幻灯片）*：顶部需提供 `[预览] / [代码]` 切换 Tab；右上角需提供 `[编辑模式切换 (手动/AI)]` 操作按钮；底部提供分页器 `[上一页/下一页/目录]`。
- **右栏 (Copilot Panel)**：`Copilot 对话窗口`。

### 3.4 侧边 AI 副驾 (The Consistent Copilot)
- 固定在屏幕右侧（Width 约占 25%-30%）。
- 具备上下文记忆，当用户在左栏切换不同的素材时，Copilot 可提供针对当前聚焦素材的专属 Quick Actions 或生成建议。
- 支持富卡片流（Rich Card Stream），例如生成封面后，能在聊天流中直接呈现带 `[全屏预览]` 的卡片交互。

## 4. 各模块适配指南 (Adapting to Other Modules)

当开发一个新的模块系统时，严格按以下映射表实现：

| 模块类型 | Tab 1 名称 (终态预览) | Tab 2 名称 (物料区) | 左侧 Menu 树 (Tab 2 下) | Detail Canvas 渲染组件要求 |
| :--- | :--- | :--- | :--- | :--- |
| **讲解视频** *(参考标杆)* | 智能课程 | 课程素材 | 原文档, 解析内容, 大纲, 脚本, 幻灯片, 语音, 封面图 | 文本流渲染 (大纲/脚本), 卡片轮播台 (幻灯片) |
| **测试问卷** | 智能问卷 | 问卷素材 | 原文档, 提炼知识点, 问卷大纲, 题库列表, 生成配置 | 题库编辑器 (单套题 CRUD), 问卷流预览引擎 |
| **研究文档** | 智能报告 | 报告素材 | 原文档, 核心洞察, 报告大纲, 章节草稿, 数据图表 | 长文编辑器 (类似 Notion), 图表参数配置板 |

## 5. UI/UX 关键交互细节卡控规范

1. **选中态对比度**：如截图所示，Tab 栏选中时字色需为纯黑 `#000000` 并加粗，未选中时为灰色。
2. **多态 Canvas 切换**：在 Tab 2 模式下选中“幻灯片”等具象元素时，画布内的二级 Tab（预览/代码）与右上角的编辑按钮（手动编辑 / ✨AI 编辑）属于**内部浮层逻辑**，不能影响外层的布局骨架。
3. **全局状态保持**：Copilot 的聊天记录与状态必须是模块级别的全局状态。在 Tab 1、Tab 2、或 Tab 2 的不同素材间切换时，聊天历史**绝对不能清空或强制刷新**。
4. **视觉连贯性**：中栏画布区统一使用浅色背景底边带微弱阴影/边框约束（区别于主视口灰色背景），确保“工作区”边界清晰。
